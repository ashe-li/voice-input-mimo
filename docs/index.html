The Loop

從口述到游標，
一條完整的 5 拍迴圈

  1. No.01 · capture

    按住 Fn 講話，放開就送出

    不用按 toggle，不用喚醒詞。按住、講、放開，三步完成輸入。

  2. No.02 · transcribe

    MiMo ASR 把中英混講真的聽懂

    用 Xiaomi 開源的 MiMo-V2.5-ASR（MLX INT4），中英 code-switching 是 native 能力， 不是「中文 ASR 強行接英文 token」。

    你說 把這個 component 重構成 hook
    傳統 ASR 把這個康波奶特 重構成 厚客
    MiMo ASR 把這個 component 重構成 hook
  3. No.03 · refine

    同一段口述，4 種輸出

    依當下需要切換：最快的純 ASR、清掉口語雜質的修正、給 Claude Code / Cursor 的英文翻譯、自動分流到模板的複合情境。

  4. No.04 · route

    關鍵字一出，模板自己接

    選「複合情境」後，純 Swift router 看內容裡有沒有「會議結論 / TODO / 驗收條件 / 敬上」這類關鍵字， 自動切到對應的 5 種模板。不打 LLM 做 routing，省一次 call。

  5. No.05 · paste

    結果回到游標原本的位置

    貼上的時機點是你放開 Fn 的當下；不會跳出獨立視窗、不會打斷你正在打字的 app。 Overlay 顯示 Listening → Refining → Done 三態，狀態列也同步。

Output Modes

4 種輸出 · 速度與用途
一目了然

中文 ASR 原文

最快

不打 LLM，講完就貼。當你只要把口述變成純文字，越快越好。

  • 適合：即時筆記、純中文輸入
  • 延遲：< 1s（音檔長度依語速）

中文 LLM 修正

快

用小 LLM call 去掉口語雜質（嗯、那個、就是…），保留原意。

  • 適合：會議發言、口齒不清的中文 cleanup
  • 不改詞意、不擴寫

英文翻譯

中

口述中文 → 英文。輸出語氣偏給 AI 工具讀的工程語，不是文學翻譯。

  • 適合：Claude Code / Cursor / 英文 PR description
  • 保留代碼識別字

中文 複合情境

中～慢

關鍵字自動分流到 5 種模板：會議紀錄 / 任務清單 / 需求草稿 / 信件 / 文章。

  • 適合：把零散口述展開成可用的初稿
  • 路由邏輯純 Swift，可預測

切換方式： Ctrl+⌥+→ / ← 在 4 種模式間輪轉， 或從 menu bar mic 圖示挑「輸出模式」子選單。

Prompts · Skills

Prompt 結構像積木
不用改程式碼

每種輸出模式背後是一組 Profile。Profile 由 N 個 Skill compose 而成 （例：output-english-only + recover-en-cn-homophones + drop-fillers）。

  • 12 個 builtin skills，可複製成自己的版本
  • 9 個 builtin profiles，三類（Refine / ClaudeCode / Structure）各自獨立
  • 三層 fallback：Profile → UserDefaults override → hardcoded builtin，任一層 miss 都不會 crash
  • JSON bundle 匯入 / 匯出，可分享給隊友或備份
Glossary

專有名詞詞庫
不再被亂翻

需要 macOS 14+（Sonoma） · 強烈推薦 Apple Silicon（MLX 跑得順） · 看所有 releases

⚠️ 還需要兩個 backend

這個 app 是前端，會打 local-llm-backend gateway（:4000）。 gateway 後面再 fan-out 到 ASR sidecar（本 repo 內附，:8766）和你自選的 LLM backend（推薦 Rapid-MLX、Ollama、vLLM 都可）。

ASR sidecar 第一次跑會下載 ~4.5 GB 的 INT4 MLX model 到 ~/.cache/mimo-asr/。LLM 自己挑，建議 Qwen3-8B 或同等級以上。
